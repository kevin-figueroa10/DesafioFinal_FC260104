PROMPT PRINCIPAL #1 - GENERACIÓN DEL CÓDIGO BASE
================================================================================

TÍTULO: Generar Sistema de Gestión de Biblioteca Universitaria en C# .NET 10

CONTENIDO DEL PROMPT:

"Genera un programa completo en C# .NET 10 para un sistema de Gestión de 
Biblioteca Universitaria en un ÚNICO archivo .cs. El programa debe incluir:

MODELOS DE DATOS:
1. Clase Libro: ID (int), Título (string), Autor (string), ISBN (string), 
   AñoPublicacion (int), Disponibles (int), Prestados (int)

2. Clase Usuario: Carné (string), Nombre (string), Carrera (string), 
   Email (string), Teléfono (string), Activo (bool)

3. Clase Préstamo: ID (int), UsuarioCarné (string), LibroID (int), 
   FechaPréstamo (DateTime), FechaDevolución (DateTime?)

FUNCIONALIDADES REQUERIDAS:

A) GESTIÓN DE LIBROS:
   - Crear nuevo libro
   - Listar todos los libros con disponibilidad
   - Actualizar información del libro
   - Eliminar libro del sistema
   - Mostrar libros disponibles
   - Validar que no haya libros duplicados por ISBN

B) GESTIÓN DE USUARIOS:
   - Registrar nuevo usuario
   - Listar todos los usuarios
   - Buscar usuario por carné
   - Activar/Desactivar usuarios
   - Validar carné con formato correcto
   - Validar email válido
   - Validar teléfono con formato

C) GESTIÓN DE PRÉSTAMOS:
   - Realizar préstamo de libro a usuario
   - Devolver libro prestado
   - Listar todos los préstamos
   - Listar préstamos activos (sin devolución)
   - Listar préstamos del usuario
   - Validar que usuario esté activo
   - Validar que haya libros disponibles
   - Validar que no tenga préstamo pendiente del mismo libro

D) PERSISTENCIA DE DATOS:
   - Cargar datos desde archivo libros.csv (formato: ID|Título|Autor|ISBN|Año|Disponibles|Prestados)
   - Cargar datos desde archivo usuarios.txt (formato: Carné|Nombre|Carrera|Email|Teléfono|Activo)
   - Cargar datos desde archivo prestamos.txt (formato: ID|UsuarioCarné|LibroID|FechaPréstamo|FechaDevolución)
   - Guardar cambios en los archivos automáticamente

E) INTERFAZ DE USUARIO:
   - Menú principal interactivo con opciones claras
   - Submenús para cada módulo (Libros, Usuarios, Préstamos)
   - Validación de entrada del usuario
   - Mensajes claros de éxito y error
   - Tablas formateadas para mostrar datos
   - Confirmación antes de operaciones destructivas

F) VALIDACIONES Y REGLAS DE NEGOCIO:
   - No permitir libros con ISBN duplicados
   - No permitir usuarios con carné duplicado
   - No permitir préstamos a usuarios inactivos
   - No permitir préstamo si no hay libros disponibles
   - Máximo 3 préstamos activos por usuario
   - Usuario solo puede tener un préstamo activo del mismo libro
   - Email debe tener formato válido (@)
   - Teléfono debe ser numérico de 8 dígitos

El código debe:
- Usar namespace 'BibliotecaUniversitaria'
- Ser completamente funcional y sin errores
- Incluir manejo de excepciones
- Validar todas las entradas del usuario
- Ser profesional y bien comentado
- Incluir una clase Program con Main que inicialice los servicios
- Incluir una clase GestorArchivos que maneje la persistencia
- Incluir clases de servicios para cada módulo
- Ser entregado en UN ÚNICO archivo .cs"

RESPUESTA GENERADA:
El modelo IA generó el archivo 'codigo_generado.cs' con aproximadamente 1200+ 
líneas de código, incluyendo todas las clases, servicios, validadores y la 
interfaz de usuario interactiva.
